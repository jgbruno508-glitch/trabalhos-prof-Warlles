2.1.1 Versões do HTML

HTML
A primeira versão, publicada em 1992 por Tim Berners-Lee, baseou-se em propostas criadas
no CERN em 1989.
HTML 4.0
Lançada em 1997, foi um marco importante por viabilizar o uso de CSS para estilização.
XHTML 1.0
Reformulou o HTML com base nas regras do XML, exigindo uma sintaxe mais rígida, como
atributos em letras minúsculas e sempre com valores.
HTML5
Lançada em 2014, substituiu as versões anteriores e introduziu novas funcionalidades e APIs da web.
Living Standard
Atualmente, o HTML é desenvolvido de forma contínua pelo WHATWG em colaboração com o W3C,
sendo tratado como uma especificação viva, sem números de versão fixos para novos desenvolvimentos.

2.1.2 Using Elements and Attributes (Uso de Elementos e Atributos)

O HTML utiliza elementos e atributos para estruturar e dar significado (semântica) ao conteúdo.

Estrutura Básica

Um documento padrão inclui o Doctype (que especifica a versão), a tag html (que engloba
todo o código), o head (para informações não visíveis, como o título da página) e o body
(onde reside todo o conteúdo exibido pelo navegador).

Formatação e Cabeçalho

Utiliza-se tags como h1 e h2 para títulos, p para parágrafos, e i ou b para
itálico e negrito, respectivamente.

Elementos:

São compostos por uma tag de abertura e uma tag de fechamento (ex: h1 e /h1), com o
conteúdo posicionado entre elas.

Atributos:

Fornecem informações adicionais aos elementos, sendo inseridos na tag de abertura no
formato nome="valor" (ex: o atributo href no elemento para definir o destino de um link).

2.1.3 Saving Web Pages as HTML Documents (Salvando Documentos HTML)

Os arquivos HTML são documentos de texto simples salvos com as extensões .html ou .htm.

processo: Criação > Edição > Salvamento > Visualização no Navegador.

Ferramentas de Desenvolvedor

A maioria dos navegadores, como o Chrome, oferece ferramentas integradas
que permitem aos desenvolvedores inspecionar e analisar o código de uma
página web diretamente.

2.2.2 Creating Lists (Listas)

O HTML suporta três tipos principais de listas: ordenadas, não ordenadas e de definição.

  • Ordenadas (ol): Listas numeradas que utilizam o elemento li para cada item
  • Não Ordenadas (ul): Listas que utilizam marcadores (pontos, quadrados, etc.) para cada item
  • De Definição (dl): Listas que associam termos com suas definições

Exemplos:

Ordenadas:
  1. Item 1
  2. Item 2
  3. Item 3
Não Ordenadas:
  • Item A
  • Item B
  • Item C

2.2.4 Including Images (Imagens)

Imagens são inseridas usando a tag img, que é um elemento vazio (não tem
tag de fechamento) e requer o atributo src para especificar o caminho da imagem.

  • inserção: O elemento img é usado para exibir imagens via atributo src
  • acessibilidade: O atributo alt é essencial para fornecer um texto
    alternativo para leitores de tela ou caso a imagem não carregue
  • Legendas: A tag figure pode ser usada para agrupar uma imagem e sua legenda,
    com a legenda definida dentro da tag figcaption

Exemplo:

Imagem

Figura com Legenda

Imagem com Tooltip

URL quebrado

O atributo src pode apontar para uma imagem local (caminho relativo) ou para uma
imagem hospedada online (URL absoluta).

2.2.5 Structure Data in Tables

Tabelas são criadas usando a tag table, com linhas definidas por tr, cabeçalhos por
th e células por td.

  • Estrutura Basica: As tabelas utilizam table, com linhas definidas por tr e
    células por td (dados) ou th (cabeçalhos).
  • Organização: para tabelas longas e recomendado usar o elemento thead para
    definir o cabeçalho da tabela tbody para o corpo da tabela e tfoot para o rodapé.
  • Mesclagem de Células: O atributo colspan permite que uma célula se estenda por várias colunas,
    enquanto rowspan permite que se estenda por várias linhas.

Exemplo:

Tabela Simples

Nome Idade
Alice 30
Bob 25

Tabela com Cabeçalho e Rodapé

Nome Idade
Alice 30
Bob 25
Total 55

2.2.6 Definig Forms (Formulários)

Formulários são criados usando a tag form, que pode conter diversos tipos de campos de entrada,
como text, password, checkbox, radio, submit, entre outros.

Base:

O elemento form é usado para criar um formulário, com o atributo action para especificar o destino
dos dados e method para definir o método de envio (GET ou POST).

Campos de Entrada:

O HTML oferece uma variedade de tipos de campos de entrada, como:

  • text: Campo de texto para entrada de dados
  • password: Campo de senha que oculta a entrada
  • checkbox: Caixa de seleção para opções múltiplas
  • radio: Botão de opção para seleção única
  • submit: Botão para enviar o formulário

Outros elementos

Incluem select para listas suspensas, textarea para textos longos, e label para rotular
campos, melhorando a acessibilidade.

dos elementos apresentados, podem ser descritos em bloco (como h1 e p) e inline (como a e img), dependendo de como

2.3 Summary and Outlook

encerra o capítulo recapitulando os conceitos fundamentais do HTML discutidos anteriormente.

2.3.1 Key points (pontos principais)

Definição:

O HTML é uma linguagem de marcação usada para definir a estrutura e a semântica das páginas web através de elementos.

Sintaxe:

Os elementos HTML consistem em uma tag de abertura e uma de fechamento.

Atributos:

Informações adicionais podem ser passadas nas tags de abertura por meio de atributos, que são compostos por um nome e
um valor associado.

Tipos de Elementos:

Existem elementos que podem conter outros (elementos filhos) ou texto, e elementos vazios, que não podem conter texto
nem outros elementos.

apartir desses conceitos básicos, o HTML oferece uma ampla variedade de elementos e atributos que permitem criar páginas web ricas e estruturadas,
e a compreensão desses fundamentos é essencial para o desenvolvimento web.
o texto recomenda algumas leitura e fala sobre as paginas não serem atraentes,
